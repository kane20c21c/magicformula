# 홀드형 vs 타이밍형 — 기술적 신호 차등적용 검증 명세

작성: 2026-06-13 · Kane & 클로이
하네스: `scripts/holdtiming_validation.py` · 그룹: `docs/holdtiming/stock_groups.csv` · 결과: `docs/holdtiming/results/`

---

## 0. 배경 / 목적

종목을 **홀드형(추세추종 — 그냥 들고 있어도 수익)** 과 **타이밍형(올라갈때·내려갈때 구분 필요)** 으로 나눴을 때
(분류 근거: 추세평활도 R²·보유수익·MDD·모멘텀갭, [[holdtype-timingtype-framework]]),
**같은 기술적 지표/와이코프 신호가 두 그룹에서 반대로 작동하는가**(홀드=지속, 타이밍=반전)를 검증한다.
검증된 항목은 그룹별 임계치·해석·매매규칙으로 반영한다. **하나씩 해결**하는 체크리스트.

### 그룹 정의 (stock_groups.csv)
- `scheme=trend3` ★ **방향중립 재분류(2026-06-13, 권장)**: 유형3 = 추세형-우상향(111)/추세형-우하향(2)/타이밍형(86). 추세성=R²≥0.7(방향무관), 방향=기울기 부호.
- `scheme=trend`  : 추세성 = 추세형(113)/타이밍형(86)
- `scheme=dir`    : 방향 = 우상향(177)/우하향(22)
- `scheme=binary` : (구) 종목구분 = 홀드(98)/타이밍(101) — 성향점수 중앙값. ⚠ BH 포함이라 방향(우상향) 오염 — V1~V4는 이 축으로 돌렸음
- `scheme=type`   : (구) Type = 홀드형(60)/타이밍형(58)/혼합형(40)/저신호(41)
- `scheme=cell`   : (구) 구획 = 섹터×종목 4셀

> ⚠ **방향중립 재분류 경위(2026-06-13)**: 구 홀드적합도가 보유수익(BH)을 포함해 "홀드형"이 사실상 **우상향 홀드**로 정의됨(Kane 지적). R²만으로 추세성을 정의하고 방향을 분리. 우하향 홀드는 현 강세장 universe에 2종목뿐(LG생활건강 R²0.85·−28%, 크래프톤 0.74·−36%, 구 분류에선 '저신호'로 오분류). V1~V4 결과는 구 binary 축이라, trend3 축 재검증 권장.

### 공통 검정 방법 (run_event_test)
조건 C 발생일 → N일 후행수익(raw, 보조 alpha) → 그룹별 중앙값·승률·n → **홀드 vs 타이밍 Mann-Whitney + 부호 분기 여부**.
기본 horizon = 5·10·20일. 최소 이벤트 수 30. 데이터: longlivevault core+extend.parquet (수급 항목은 core 한정).

### 성공/지지 기준 (공통 템플릿)
1. 그룹 간 후행수익 중앙값 **부호가 갈림**(diverge_sign=True) 또는 Mann-Whitney p<0.05 로 유의차.
2. 방향이 가설(expect)과 일치.
3. 강건성: 5·10·20일 중 2개 이상에서 동일 패턴.
4. (보조) baseline(무조건 후행수익) 대비 조건부가 개선/악화되는 방향 확인.
> ⚠ 강세장 기저효과로 raw 후행수익은 전반적으로 +편향 → **그룹 간 상대차**로 판단(절대 부호 맹신 금지). alpha 모드 병기.

## Phase 1.5 — 국면지표×그룹 속도 정합 (M 시리즈)

설계 의도상 **Wyckoff_Phase는 느리게(서술·하락검출 17~18일), Wyckoff_Label은 민감하게(하락검출 4~5일)** 작동한다([[holdtype-timingtype-framework]], phase.py/classifier.py).
→ **느린 Phase는 홀드셀, 민감한 Label은 타이밍셀에 맞을 것**이라는 가설.

| ID | 항목 | 가설 | 방법 | 상태 |
|---|---|---|---|---|
| M1 | 하락신호 청산품질: Phase vs Label × 그룹 | **H1 홀드↔Phase**: 홀드형은 Phase 청산엣지>Label(Label 휩쏘 多). **H2 타이밍↔Label**: 타이밍형은 Label 적시·정합>Phase(Phase 너무 느려 반등 놓침) | 하네스 `run_m1` (`--item M1`) | **H1 ✘기각 / H2 △약지지** |
| M2 | 상승국면 상태매수 유효성: Markup/Uptrend × 그룹 | 상승국면 보유 매수가 baseline 대비 +엣지인가? (추세형 유효 / 타이밍형 역효과 예상) | 조건 `c_markup_state`/`c_uptrend_state`, 매수엣지=상태med−그룹baseline | **✘ 무효**(추세형 중립·타이밍 역효과) |

- **신호 정의**: Phase_down = Wyckoff_Phase Downtrend 진입(전일≠), Label_down = Wyckoff_Label Markdown 진입(변형 c_label_down2 = Markdown∪Distribution).
- **지표(그룹×신호)**: 발화빈도(events/obs), 신호후 후행수익 중앙값, **휩쏘율 P(fwd>0|신호)**(낮을수록 좋은 청산), **청산엣지 = 그룹 baseline 중앙값 − 신호 중앙값**(+면 약세 예고). horizon 5·10·20.
- **예측(지지 시)**: 홀드그룹 → Phase 청산엣지 > Label, Label 휩쏘율 높음. 타이밍그룹 → Label 엣지·적시성(고점후 발화 빠름) > Phase.
- **강세장 보정**: 절대 부호 아닌 **그룹 baseline 대비 상대차**로 판단. 선택편향(홀드=사후승자) 잔존 → "그룹 간 상대 정합"으로만 해석.
- 보조: 발화빈도/적시성(직전 고점 후 신호일수)으로 "민감 vs 느림" 직접 확인.
